## Critical Technical Requirements

### 1. Responsive Design (MANDATORY)
```
Mobile (default): Single column, stacked content, py-8 to py-12 spacing
Tablet (md:): 2 columns where appropriate, py-12 to py-16 spacing
Desktop (lg:): Full layouts, 3-4 columns, py-16 to py-24 spacing
```

### 2. Every Section Must Include:
- Container wrapper: `<div class="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">`
- Proper section spacing: `py-16 md:py-20 lg:py-24`
- Heading hierarchy: h1 (hero only), h2 (section titles), h3 (subsections)

### 3. Hero Section Requirements (CRITICAL):
```html
<section class="min-h-screen flex items-center bg-[colors]">
  <div class="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 w-full">
    <div class="text-center lg:text-left lg:grid lg:grid-cols-2 gap-12 items-center">
      <div>
        <h1 class="text-5xl md:text-6xl lg:text-7xl font-bold mb-6">
          [Benefit-Focused Headline]
        </h1>
        <p class="text-xl md:text-2xl mb-8 text-gray-600">
          [Clear value proposition addressing target audience pain point]
        </p>
        <div class="flex flex-col sm:flex-row gap-4">
          <a href="#" class="px-8 py-4 bg-[primary] text-white rounded-lg text-lg font-semibold hover:bg-[primary-dark] transition">
            [Action-Oriented CTA]
          </a>
        </div>
      </div>
      <div class="hidden lg:block">
        <img src="[unsplash image]" alt="[descriptive alt]" class="rounded-lg shadow-2xl">
      </div>
    </div>
  </div>
</section>
```

### 4. Feature Cards Pattern:
```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">
  <div class="p-6 rounded-xl bg-white shadow-lg hover:shadow-xl transition">
    <div class="w-12 h-12 bg-[color] rounded-lg flex items-center justify-center mb-4">
      <!-- Icon/Number -->
    </div>
    <h3 class="text-xl font-bold mb-2">[Feature Title]</h3>
    <p class="text-gray-600">[Feature description focusing on benefit]</p>
  </div>
</div>
```

### 5. Call-to-Action Buttons:
```html
<a href="#" class="inline-flex items-center px-8 py-4 bg-[primary] text-white rounded-lg text-lg font-semibold hover:bg-[primary-dark] transform hover:scale-105 transition duration-200 shadow-lg">
  [Action Text]
</a>
```

## Content Quality Standards (MANDATORY)

### Headlines Must:
- Lead with benefit, not feature
- Be specific and quantifiable when possible
- Address target audience directly
- Create urgency or curiosity

❌ Bad: "Our Software Platform"
✅ Good: "Double Your Productivity in 30 Days"

### Copy Must:
- Focus on outcomes and transformations
- Use active voice
- Be scannable (short paragraphs, bullets)
- Include social proof where relevant
- Address objections preemptively

### CTA Buttons Must:
- Use action verbs
- Create urgency or value
- Be specific about what happens next

❌ Bad: "Submit" "Learn More"
✅ Good: "Start Free Trial" "Get Instant Access" "Book My Spot"

## Image Requirements

Use high-quality Unsplash images relevant to the business niche:
- Hero: `https://images.unsplash.com/photo-[relevant]?w=1200&q=80`
- Features: `https://images.unsplash.com/photo-[relevant]?w=600&q=80`
- Testimonials: `https://images.unsplash.com/photo-[headshot]?w=200&q=80`

Always include descriptive alt text.

## Output Format (STRICT)

Return ONLY valid, complete HTML. No markdown, no code blocks, no explanations:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[Business Name] | [Key Benefit]</title>
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body class="antialiased">
  <!-- All sections here -->
</body>
</html>
```
